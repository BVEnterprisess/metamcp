**Cursor** is a popular AI-powered code editor that supports MCP (Model Context Protocol) integration. This guide shows you how to connect Cursor to your MetaMCP endpoints for enhanced coding capabilities.

## Prerequisites

Before starting, ensure you have:

- **Cursor IDE** installed and running
- **MetaMCP** running locally or deployed
- **Active endpoint** configured in MetaMCP
- **API key** generated (if authentication is enabled)

## Basic Configuration

### Simple mcp.json Setup

Create or edit your `mcp.json` file in Cursor's configuration directory:

<CodeGroup>
```json Basic Configuration
{
  "mcpServers": {
    "MetaMCP": {
      "url": "http://localhost:12008/metamcp/your-endpoint-name/sse"
    }
  }
}
```

```json With Authentication
{
  "mcpServers": {
    "MetaMCP": {
      "url": "http://localhost:12008/metamcp/your-endpoint-name/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_your_api_key_here"
      }
    }
  }
}
```

```json Multiple Endpoints
{
  "mcpServers": {
    "MetaMCP-Dev": {
      "url": "http://localhost:12008/metamcp/dev-tools/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_dev_key"
      }
    },
    "MetaMCP-Research": {
      "url": "http://localhost:12008/metamcp/research-tools/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_research_key"
      }
    }
  }
}
```
</CodeGroup>

### Configuration File Location

The `mcp.json` file should be placed in Cursor's configuration directory:

<CodeGroup>
```bash macOS
~/Library/Application Support/Cursor/User/mcp.json
```

```bash Windows
%APPDATA%\Cursor\User\mcp.json
```

```bash Linux
~/.config/Cursor/User/mcp.json
```
</CodeGroup>

## Configuration Options

### Transport Types

MetaMCP supports different transport protocols. **SSE is recommended** for Cursor:

<AccordionGroup>
  <Accordion icon="broadcast-tower" title="SSE (Recommended)">
    **Server-Sent Events** provide real-time communication:
    
    ```json
    {
      "mcpServers": {
        "MetaMCP": {
          "url": "http://localhost:12008/metamcp/dev-tools/sse"
        }
      }
    }
    ```
    
    **Benefits:**
    - Real-time updates
    - Automatic reconnection
    - Native browser support
    - Low latency
  </Accordion>

  <Accordion icon="http" title="Streamable HTTP (Alternative)">
    **HTTP-based communication** for compatibility:
    
    ```json
    {
      "mcpServers": {
        "MetaMCP": {
          "url": "http://localhost:12008/metamcp/dev-tools/mcp"
        }
      }
    }
    ```
    
    **Note:** May have higher latency than SSE
  </Accordion>
</AccordionGroup>

### Authentication Methods

<AccordionGroup>
  <Accordion icon="key" title="API Key Authentication">
    **Most common method** using Authorization header:
    
    ```json
    {
      "mcpServers": {
        "MetaMCP": {
          "url": "http://localhost:12008/metamcp/dev-tools/sse",
          "headers": {
            "Authorization": "Bearer sk_mt_your_key_here"
          }
        }
      }
    }
    ```
    
    <Warning>
      Never commit API keys to version control. Use environment variables or secure configuration methods.
    </Warning>
  </Accordion>

  <Accordion icon="unlock" title="No Authentication">
    **For public endpoints** without authentication:
    
    ```json
    {
      "mcpServers": {
        "MetaMCP": {
          "url": "http://localhost:12008/metamcp/public-tools/sse"
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Advanced Configuration

### Custom Headers

Add custom headers for enhanced functionality:

```json
{
  "mcpServers": {
    "MetaMCP": {
      "url": "http://localhost:12008/metamcp/dev-tools/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_your_key",
        "X-Client": "Cursor",
        "X-Version": "1.0.0",
        "User-Agent": "Cursor/MetaMCP-Integration"
      }
    }
  }
}
```

### Environment-Specific Configuration

Use different configurations for different environments:

<CodeGroup>
```json Development
{
  "mcpServers": {
    "MetaMCP-Local": {
      "url": "http://localhost:12008/metamcp/dev-tools/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_dev_key"
      }
    }
  }
}
```

```json Production
{
  "mcpServers": {
    "MetaMCP-Prod": {
      "url": "https://metamcp.yourcompany.com/metamcp/dev-tools/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_prod_key"
      }
    }
  }
}
```
</CodeGroup>

### Timeout and Retry Configuration

Configure connection timeouts and retry behavior:

```json
{
  "mcpServers": {
    "MetaMCP": {
      "url": "http://localhost:12008/metamcp/dev-tools/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_your_key"
      },
      "timeout": 30000,
      "retries": 3,
      "retryDelay": 1000
    }
  }
}
```

## Testing Your Configuration

### Verify Connection

1. **Save your mcp.json** file in the correct location
2. **Restart Cursor** to load the new configuration
3. **Open the MCP panel** in Cursor (usually in the side panel)
4. **Check connection status** - should show "Connected"
5. **Browse available tools** to verify functionality

### Test Tool Execution

1. **Select a tool** from the MCP panel
2. **Provide required parameters**
3. **Execute the tool** and verify results
4. **Check for errors** in the Cursor console

### Debug Connection Issues

If you encounter connection problems:

<AccordionGroup>
  <Accordion icon="warning" title="Connection Failed">
    **Check these common issues:**
    
    1. **URL correctness**: Verify endpoint URL and name
    2. **MetaMCP running**: Ensure MetaMCP is accessible
    3. **API key validity**: Check if API key is correct and active
    4. **Network connectivity**: Test URL in browser
    5. **Firewall settings**: Ensure ports are open
  </Accordion>

  <Accordion icon="shield" title="Authentication Errors">
    **Authentication troubleshooting:**
    
    1. **API key format**: Should start with `sk_mt_`
    2. **Header format**: Use `Authorization: Bearer {key}`
    3. **Key permissions**: Verify key has required scopes
    4. **Key expiration**: Check if API key is still valid
  </Accordion>

  <Accordion icon="slow" title="Performance Issues">
    **Optimization strategies:**
    
    1. **Use SSE transport**: Better performance than HTTP
    2. **Local deployment**: Reduce network latency
    3. **Optimize namespace**: Reduce number of tools
    4. **Check MetaMCP logs**: Look for performance bottlenecks
  </Accordion>
</AccordionGroup>

## Production Deployment

### Secure Configuration

For production use, follow security best practices:

```json
{
  "mcpServers": {
    "MetaMCP-Prod": {
      "url": "https://metamcp.yourcompany.com/metamcp/prod-tools/sse",
      "headers": {
        "Authorization": "Bearer sk_mt_prod_key_with_limited_scope"
      }
    }
  }
}
```

**Security checklist:**
- ✅ Use HTTPS in production
- ✅ Rotate API keys regularly
- ✅ Use least-privilege API keys
- ✅ Monitor API key usage
- ✅ Implement rate limiting
- ✅ Use environment-specific keys

### Team Configuration

For team environments, consider:

<CodeGroup>
```json Shared Configuration Template
{
  "mcpServers": {
    "MetaMCP-Team": {
      "url": "https://metamcp.company.com/metamcp/team-tools/sse",
      "headers": {
        "Authorization": "Bearer ${METAMCP_API_KEY}",
        "X-Team": "${TEAM_NAME}",
        "X-User": "${USER_ID}"
      }
    }
  }
}
```

```bash Environment Variables
# Set in your shell profile or CI/CD
export METAMCP_API_KEY=sk_mt_your_team_key
export TEAM_NAME=engineering
export USER_ID=john.doe
```
</CodeGroup>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion icon="file" title="Configuration File Issues">
    **File location problems:**
    
    - Verify `mcp.json` is in correct directory
    - Check file permissions (should be readable)
    - Ensure valid JSON syntax
    - Restart Cursor after changes
  </Accordion>

  <Accordion icon="network" title="Network Connectivity">
    **Connection troubleshooting:**
    
    ```bash
    # Test MetaMCP endpoint directly
    curl -H "Authorization: Bearer sk_mt_your_key" \
      http://localhost:12008/metamcp/dev-tools/sse
    
    # Check if endpoint is accessible
    curl -I http://localhost:12008/health
    ```
  </Accordion>

  <Accordion icon="bug" title="Tool Execution Errors">
    **When tools fail to execute:**
    
    1. Check tool parameters are correct
    2. Verify tool is enabled in namespace
    3. Check MetaMCP server logs
    4. Test tool in MetaMCP inspector
    5. Verify middleware isn't blocking requests
  </Accordion>
</AccordionGroup>

### Debug Mode

Enable debug logging in Cursor:

1. **Open Cursor settings**
2. **Search for "MCP"**
3. **Enable debug logging**
4. **Check console** for detailed error messages
